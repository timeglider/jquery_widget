The Timeglider jQuery Plugin

This is a prototype of a jQuery plugin for Timeglider. It's still got lots of scaffolding, is fairly buggy — but you get the idea of what's possible. You can see our current (Flash-based) system here. Still to come in this jQuery version: event spans, legends, timezones, images, and more. This currently seems to work on all Mac browsers, IE8, and Windows Firefox. More extensive browser testing to come. as well as updating to new jQuery/jQuery UI

Some goals:

  • images and image albums presented as events
  • color settings for event spans
  • multiple options for modal windows
  • multiple file links, i.e. with audio, video etc.
  • interpreting existing HTML as a simple alternative to JSON
  • importers for event-oriented APIs
  • full gesture actions on ipads and other larger mobile devices

The code

For developers/designers, the jQuery code will be very simple: 

<!-- html: -->
   <div id='placement'></div>
    
<!-- js: -->
   $(document).ready(function () { 

      var tg1 = $("#placement").timeglider({
         "max_zoom":60, 
         "backgroundColor":"#F3F3F3", 
         "initial_timeline_id": "jshist",
         "data_source":"json_tests/js_history.json"
     });
   });
Using CSS, one can style/size the "placement" div, also. The jQuery setup (starting with the dollar sign) indicates some of the initial settings; there will also be public methods so that the reference (in this case "tg1" can use methods like "zoomTo()" or "gotoDate()", etc. Data sources will use either JSON or XML; eventually we'll provide a way to take an HTML table of dates/events/etc, and transform it into a viewer instance.

The general plan is to offer this plugin as a free and open-source resource, but for sites publishing at high volume, and/or where it is used as part of a commercial enterprise, a commercial license will apply. The ultimate goal is to have a thriving ecosystem module, context, and other code development.

There are many possibilities.
